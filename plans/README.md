## Findings considered and rejected

(Recorded so the next audit doesn't re-litigate them.)

- **Oversell race + non-atomic payment confirmation (legacy checkout)** — documented, root-caused, and deliberately deferred into the active reservation cutover (docs/plans/2026-06-checkout-reservation-rebuild.md). Not re-planned.
- **Fee divergence between `apps/web/src/lib/fees.ts` (tax-on-fee) and `packages/api` calculator** — intentional pending Stage-3 cutover, per the code comment and PR #304. Tracked there.
- **Direct Prisma usage in ~22 route files / 600-line initiate route / type mirroring with the Expo app** — the entire shared-packages platform plan exists for this (docs/plans/2026-06-shared-packages-platform.md). Not re-planned.
- **"isDraft hardcoded false" (organizer events route)** — false alarm: `isDraft: false` in a Prisma `select` merely omits the field. Possibly a typo for `true`; cosmetic; fold into any future touch of that route.
- **"$0 checkout for nonexistent ticket type"** — false alarm: `validateTicketType(null, …)` returns `validatedQuantity: 0`, so no ticket is sold.
- **"Webhook missing signature verification"** — false alarm: `stripe.webhooks.constructEvent` is called with the endpoint secret (webhook.ts:46).
- **`isPlatformOwner` suffix-bypass concern** — `email.endsWith('@usetroptix.com')` cannot be spoofed by subdomain tricks; plan 014 adds the test that documents this.
- **Speculative auth hardening items** (claim validation, trigger email-case theory, webhook metadata cross-check) — no demonstrated path; not worth plans.
- **Extracting the 2,700-line privacy-policy TSX / CMS for legal pages** — real smell, no concrete cost; not worth doing now.
- **Hardcoded `America/New_York` in date helpers** — plausibly intentional for the current market; revisit when a non-US-Eastern event exists.
- **Pagination on organizer lists** — real at scale, but invasive UI work owned by the upcoming redesign; premature as a standalone plan.
- **Stripe SDK v14 lag / `micro` dependency / Pages-Router webhook migration** — all subsumed by the checkout cutover's new App Router webhook; upgrading separately first is throwaway work.
- **Tailwind v4 / antd→shadcn convergence / dark mode** — already decided and planned (ADR 0001/0002, design-system plan).
- **Checkout-config client-fetch waterfall** — real, but that page is being redesigned (cutover client half); fixing the old container is throwaway.
- **Duplicate-email idempotency via an `emailSentAt` column** — superseded by local plan 005's approach, which prevents the duplicate at the source.
